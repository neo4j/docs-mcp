[[installation]]
= Installation
:description: This section describes the installation of Neo4j MCP.

Visit the release page for Neo4j MCP: https://github.com/neo4j/mcp/releases

. Download the archive for your operating system.
. Extract and place `neo4j-mcp` in a directory present in your PATH variables.

Mac and Linux:

[source, bash]
----
chmod +x neo4j-mcp
sudo mv neo4j-mcp /usr/local/bin/
----

Windows (PowerShell or cmd):

[source, shell]
----
move neo4j-mcp.exe C:\Windows\System32
----

Verify the neo4j-mcp installation:

[source, bash]
----
neo4j-mcp -v
----

This should print the installed version.


== Configure VSCode (MCP)

=== STDIO mode configuration

Create / edit `mcp.json`:

[source, json]
----
{
  "servers": {
    "neo4j": {
      "type": "stdio",
      "command": "neo4j-mcp",
      "env": {
        "NEO4J_URI": "bolt://localhost:7687",
        "NEO4J_USERNAME": "neo4j",
        "NEO4J_PASSWORD": "password",
        "NEO4J_DATABASE": "neo4j",
        "NEO4J_READ_ONLY": "true"
      }
    }
  }
}
----

Set environment variables in the `env` object as shown above.

Alternatively, you can use command-line flags in the `args` array to override environment variables:

[source, json]
----
{
  "servers": {
    "neo4j": {
      "type": "stdio",
      "command": "neo4j-mcp",
      "args": [
        "--neo4j-uri", "bolt://localhost:7687",
        "--neo4j-username", "neo4j",
        "--neo4j-password", "password",
        "--neo4j-database", "neo4j"
      ],
      "env": {
        "NEO4J_READ_ONLY": "true"
      }
    }
  }
}
----

=== HTTP mode configuration

For HTTP mode with CORS support:

[source, json]
----
{
  "servers": {
    "neo4j-http": {
      "type": "http",
      "command": "neo4j-mcp",
      "env": {
        "NEO4J_URI": "bolt://localhost:7687",
        "NEO4J_MCP_TRANSPORT": "http",
        "NEO4J_MCP_HTTP_HOST": "127.0.0.1",
        "NEO4J_MCP_HTTP_PORT": "8080",
        "NEO4J_MCP_HTTP_ALLOWED_ORIGINS": "http://localhost:3000,https://app.example.com"
      }
    }
  }
}
----

For HTTP mode with TLS enabled:

[source, json]
----
{
  "servers": {
    "neo4j-https": {
      "type": "http",
      "command": "neo4j-mcp",
      "env": {
        "NEO4J_URI": "bolt://localhost:7687",
        "NEO4J_MCP_TRANSPORT": "http",
        "NEO4J_MCP_HTTP_HOST": "0.0.0.0",
        "NEO4J_MCP_HTTP_PORT": "8443",
        "NEO4J_MCP_HTTP_TLS_ENABLED": "true",
        "NEO4J_MCP_HTTP_TLS_CERT_FILE": "/path/to/cert.pem",
        "NEO4J_MCP_HTTP_TLS_KEY_FILE": "/path/to/key.pem"
      }
    }
  }
}
----

[NOTE]
====
In HTTP mode, credentials are provided per-request via Basic Authentication headers.
Do NOT set `NEO4J_USERNAME` or `NEO4J_PASSWORD` environment variables.
====

Restart VSCode.
Open Copilot Chat and make sure that the prompt "List Neo4j MCP tools" returns expected results.

Also refer to link:https://code.visualstudio.com/docs/copilot/customization/mcp-servers[VSCode Copilot docs].


== Configure Claude desktop

Install link:https://claude.ai/download[Claude for desktop].

Configure Claude for desktop for all MCP servers you want to use.
Open your Claude for desktop App configuration in a text editor.
Find the files at:

- (MacOS/Linux) `~/Library/Application Support/Claude/claude_desktop_config.json`
- (Windows) `$env:AppData\Claude\claude_desktop_config.json`

Make sure to create the file if it doesn't exist.

=== STDIO mode configuration

Add the `neo4j-mcp` MCP in the `mcpServers` key:

[source, json]
----
{
  "mcpServers": {
    "neo4j-mcp": {
      "type": "stdio",
      "command": "neo4j-mcp",
      "args": [],
      "env": {
        "NEO4J_URI": "bolt://localhost:7687",

----

For HTTP mode with TLS enabled:

[source, json]
----
{
  "mcpServers": {
    "neo4j-mcp-https": {
      "type": "http",
      "command": "neo4j-mcp",
      "args": [],

        "NEO4J_URI": "bolt://localhost:7687",

        "NEO4J_MCP_HTTP_TLS_KEY_FILE": "/path/to/key.pem"
      }
    }
  }

----

Notes:

- Command-line flags (in `args`) take precedence over environment variables (in `env`).
- `NEO4J_READ_ONLY` is optional. Set to `true` to disable write tools.
- Neo4j Desktop example URI: `bolt://localhost:7687`.
- For Aura, use the connection string from the Aura Console.
- **STDIO mode**: All connection parameters (URI, username, password) are required and have no default values.
- **HTTP mode**: Only the URI is required. Credentials are provided per-request via Basic Authentication headers. Do NOT set `NEO4J_USERNAME` or `NEO4J_PASSWORD` in HTTP mode.
